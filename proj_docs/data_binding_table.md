# Схема привязки данных к ячейкам гексагональной сетки

## Обзор

В данном документе описаны типы данных, которые привязываются к ячейкам H3 гексагональной сетки для модели оценки риска оползней, а также методы привязки.

---

## 1. Текущие атрибуты ячейки

Каждая ячейка H3 (resolution 9) имеет следующий набор атрибутов:

### Идентификация

| Атрибут | Тип | Описание | Пример |
|---------|-----|----------|--------|
| `cell_id` | string | H3 идентификатор ячейки | `891969a6c3fffff` |
| `h3_resolution` | integer | Уровень разрешения H3 | `9` |
| `district_id` | integer | Код района (1--4) или NULL | `1` |
| `district_name` | string | Название района | `Адлерский внутригородской район` |

### Координаты центроида

| Атрибут | Тип | Описание | Система |
|---------|-----|----------|---------|
| `centroid_lon` | float | Долгота центроида | WGS84 |
| `centroid_lat` | float | Широта центроида | WGS84 |
| `centroid_x_utm` | float | X центроида (метры) | UTM 37N |
| `centroid_y_utm` | float | Y центроида (метры) | UTM 37N |

### Геометрия

| Атрибут | Тип | Описание | Значение |
|---------|-----|----------|----------|
| `area_m2` | float | Площадь ячейки в м² | ~105 000 |
| `area_km2` | float | Площадь ячейки в км² | ~0.105 |
| `coverage_pct` | float | Процент покрытия территорией (0--100) | 0--100 |
| `hex_side_m` | float | Примерная длина стороны шестиугольника | ~196 |

### Визуализация

| Атрибут | Тип | Описание |
|---------|-----|----------|
| `color` | string | HEX-код цвета по району |

## 2. Привязка данных из рельефа (DEM)

Для извлечения признаков рельефа цифровая модель рельефа (растр) агрегируется по полигону каждой ячейки методом зональной статистики.

| Признак | Описание | Метод агрегации |
|---------|----------|-----------------|
| `elev_mean` | Средняя высота | Среднее по пикселям в ячейке |
| `elev_min` | Минимальная высота | Минимум по пикселям |
| `elev_max` | Максимальная высота | Максимум по пикселям |
| `elev_range` | Перепад высот | `elev_max - elev_min` |
| `elev_std` | Разброс высот | Стандартное отклонение |
| `slope_mean` | Средний уклон | Среднее по растру уклонов |
| `slope_max` | Максимальный уклон | Максимум по растру уклонов |
| `aspect_mode` | Преобладающая экспозиция | Мода (наиболее частое направление) |
| `curvature_mean` | Средняя кривизна | Среднее по растру кривизны |
| `twi_mean` | Индекс влажности (TWI) | Среднее по растру TWI |

Процесс: растр перепроецируется в UTM Zone 37N, вычисляются производные растры (уклон, экспозиция, кривизна, TWI), затем для каждого полигона ячейки вычисляется зональная статистика.

---

## 3. Привязка данных об оползнях

Данные об оползнях представлены точками с координатами. К каждой ячейке привязываются следующие признаки:

| Признак | Описание | Метод агрегации |
|---------|----------|-----------------|
| `ls_count` | Количество оползней в ячейке | Подсчёт точек внутри полигона |
| `ls_has_event` | Был ли оползень | 1 если `count > 0`, иначе 0 |
| `ls_nearest_m` | Расстояние до ближайшего оползня | Расстояние от центроида до ближайшей точки |
| `ls_density_5km` | Плотность в радиусе 5 км | Подсчёт в буфере / площадь буфера |

Процесс: точки перепроецируются в UTM 37N, выполняется пространственное соединение (`sjoin`) для подсчёта точек в ячейке, строится KD-дерево для поиска ближайшей точки, создаётся буфер радиусом 5 км вокруг центроида для расчёта плотности.

---

## 4. Привязка сейсмических данных

Сейсмические данные представлены точками с координатами, магнитудой и датой. К каждой ячейке привязываются следующие признаки:

| Признак | Описание | Метод агрегации |
|---------|----------|-----------------|
| `eq_count_50km` | Количество землетрясений в радиусе 50 км | Подсчёт в буфере вокруг центроида |
| `eq_max_mag_50km` | Максимальная магнитуда в радиусе 50 км | Максимум магнитуды в буфере |
| `eq_nearest_km` | Расстояние до ближайшего землетрясения | Расстояние от центроида |

Процесс: создаётся буфер радиусом 50 км вокруг центроида каждой ячейки, подсчитывается количество точек в буфере, определяется максимальная магнитуда, вычисляется расстояние до ближайшей точки.

---

## 5. Привязка спутниковых данных

Спутниковые данные (NDVI и классификация земного покрова) агрегируются методом зональной статистики по полигону ячейки.

| Признак | Описание | Метод агрегации |
|---------|----------|-----------------|
| `ndvi_mean` | Средний вегетационный индекс NDVI | Среднее по пикселям в ячейке |
| `ndvi_std` | Разброс NDVI | Стандартное отклонение |
| `bare_soil_pct` | Доля голой почвы | Процент пикселей без растительности |

---

## 6. Привязка погодных данных

Данные об осадках привязываются к центроидам ячеек методом пространственной интерполяции.

| Признак | Описание | Метод агрегации |
|---------|----------|-----------------|
| `precip_7d` | Осадки за 7 дней | Интерполяция на центроид |
| `precip_30d` | Осадки за 30 дней | Интерполяция на центроид |
| `precip_90d` | Осадки за 90 дней | Интерполяция на центроид |

Процесс: данные интерполируются (IDW, Kriging или извлечение из растра) на координаты центроидов ячеек, агрегируются за соответствующие периоды.

## 8. Методы привязки

### Зональная статистика (растровые данные)

Применяется для данных рельефа и спутниковых снимков. Для каждого полигона ячейки вычисляются статистики (среднее, минимум, максимум, стандартное отклонение) по всем пикселям растра, попадающим внутрь полигона.

### Пространственное соединение (точечные данные)

Применяется для подсчёта точек внутри ячейки. Точки с координатами привязываются к ячейке, если попадают в её полигон (предикат `within`).

### Буферный анализ

Применяется для расчёта плотности и статистики в заданном радиусе. Вокруг центроида ячейки создаётся буфер (5 км для оползней, 50 км для землетрясений), внутри которого подсчитываются точки и вычисляются агрегированные значения.

### Расстояние до ближайшего объекта

Применяется для определения близости к оползням и землетрясениям. Строится KD-дерево по координатам точек, для каждого центроида ячейки находится ближайшая точка и вычисляется расстояние.

### Пространственная интерполяция

Применяется для погодных данных. Значения от станций или растровых продуктов интерполируются на координаты центроидов ячеек (IDW, Kriging).
