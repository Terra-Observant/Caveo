- Расположен в горно-лесистой местности

## 1.2 Система индексации H3

Для пространственного анализа используется система **H3 Hierarchical Hexagonal Geospatial Index**, разработанная компанией Uber.

### Что такое H3

H3 -- это глобальная система дискретизации земной поверхности на правильные шестиугольники. Каждая ячейка H3 имеет уникальный идентификатор (16-значное шестнадцатеричное число), который позволяет:

- Эффективно индексировать пространственные данные
- Выполнять быстрые пространственные операции
- Легко масштабировать сетку (переход между уровнями разрешения)
- Соседство ячеек вычисляется без геометрических операций

### Почему H3

| Преимущество | Описание |
|-------------|----------|
| **Равноудалённые соседи** | У каждого шестиугольника 6 соседей на одинаковом расстоянии, в отличие от квадратных сеток. |
| **Единый стандарт** | H3 используется в глобальных проектах Uber, Lyft, Netflix, NASA и других компаниях. |
| **Иерархичность** | 16 уровней разрешения позволяют работать с масштабами от глобального до локального. |
| **Эффективность** | Операции соседства и агрегации выполняются через арифметику идентификаторов, без геометрических вычислений. |
| **Отсутствие искажений** | Ячейки H3 имеют почти одинаковую площадь по всему земному шару, в отличие от широтно-долготных сеток. |
| **Поддержка** | Библиотека H3 доступна для Python, JavaScript, Java, C++, R и других языков. |

### Иерархия разрешений H3

H3 имеет 16 уровней разрешения (resolution 0-15). С увеличением разрешения площадь ячейки уменьшается примерно в 7 раз:

| Resolution | Средняя площадь ячейки | Примерное применение |
|-----------|----------------------|---------------------|
| 5 | ~252 км² | Глобальный масштаб |
| 6 | ~36 км² | Региональный масштаб |
| 7 | ~5.1 км² | Городской масштаб |
| **8** | **~0.73 км²** | Районный масштаб |
| **9** | **~0.105 км²** | **Локальный масштаб (выбран для проекта)** |
| 10 | ~0.015 км² | Детальный масштаб |
| 11 | ~0.002 км² | Сверхдетальный масштаб |

### Выбранный уровень: Resolution 9

Для проекта выбран **H3 resolution 9**, что обеспечивает:

| Параметр | Значение |
|----------|----------|
| Средняя площадь ячейки | ~0.105 км² (~105 000 м²) |
| Средний диаметр ячейки | ~370 м |
| Идентификатор ячейки | 15-значное шестнадцатеричное число (например, `891969a6c3fffff`) |
| Общее количество ячеек для Большого Сочи | 34 070 (после фильтрации 33 555 привязанных) |

Данный уровень разрешения выбран исходя из:
- Достаточной детализации для локального анализа оползневой опасности
- Вычислительной эффективности при работе с большими объёмами данных
- Близости площади ячейки к исходному требованию (~0.1 км²)
- Распространённости данного уровня в практических приложениях H3

### Фильтрация ячеек

На начальном этапе генерируются все H3 ячейки, покрывающие bounding box территории. Затем ячейки фильтруются по следующим критериям:

- Центроид ячейки находится внутри границ Большого Сочи **ИЛИ**
- Ячейка пересекается с границей минимум на **10%** своей площади

Добавлен атрибут `coverage_pct` -- процент площади ячейки, покрытый территорией.

## 1.3 Система координат

### Рабочая система координат

**UTM Zone 37N (EPSG:32637)** -- для всех метрических расчётов:
- Расчёт площадей
- Расчёт расстояний
- Зональная статистика

Преимущества:
- Метрическая система (метры)
- Минимальные искажения в пределах зоны
- Подходит для Сочи (зона 37N, longitude 36°E--42°E)

### Система хранения и визуализации

**WGS84 (EPSG:4326)** -- для хранения и визуализации:
- Хранение геоданных (GeoJSON)
- Визуализация на картах
- Совместимость с внешними источниками
- Широты/долготы в градусах

### H3 и системы координат

Ячейки H3 хранятся в географических координатах (широта/долгота, WGS84). Для метрических расчётов (площади, расстояния) геометрии ячеек перепроецируются в UTM Zone 37N.

## 1.4 Привязка ячеек к районам

Привязка осуществляется по **положению центроида** ячейки H3:

1. Вычисляется центроид шестиугольника (координаты lat/lng)
2. Определяется, в каком районе находится центроид (пространственное соединение `sjoin` с предикатом `within`)
3. Если центроид попадает в границы района -- ячейке присваивается его `district_id` и `district_name`
4. Если центроид вне всех районов -- присваивается `NULL`

Использование центроида обеспечивает однозначную привязку каждой ячейки к одному району, избегая дублирования ячеек на границах.

### Цветовая схема

Каждому району назначен уникальный цвет для визуализации:

| Район | Цвет | HEX-код |

| Центральный | Оранжевый | `#e67e22` |
| Лазаревский | Фиолетовый | `#9b59b6` |
| Не определён | Серый | `#95a5a6` |

## 2.2 Привязка растровых данных (DEM, спутниковые данные)

Для растровых данных применяется **зональная статистика** -- вычисление статистик по всем пикселям растра, попадающим в полигон ячейки H3.

### Признаки из DEM

| Признак | Описание | Метод агрегации |
|---------|----------|-----------------|
| `elev_mean` | Средняя высота | Среднее по пикселям в ячейке |
| `elev_min` | Минимальная высота | Минимум по пикселям |
| `elev_max` | Максимальная высота | Максимум по пикселям |
| `elev_range` | Перепад высот | `elev_max - elev_min` |
| `elev_std` | Разброс высот | Стандартное отклонение |
| `slope_mean` | Средний уклон | Среднее по растру уклонов |
| `slope_max` | Максимальный уклон | Максимум по растру уклонов |
| `aspect_mode` | Преобладающая экспозиция | Мода (наиболее частое направление) |
| `curvature_mean` | Средняя кривизна | Среднее по растру кривизны |
| `twi_mean` | Индекс влажности (TWI) | Среднее по растру TWI |

Рекомендуемые источники DEM: SRTM (30 м), ALOS World 3D (30 м), Copernicus DEM (30 м).

### Признаки из спутниковых данных

| Признак | Описание | Метод агрегации |
|---------|----------|-----------------|
| `ndvi_mean` | Вегетационный индекс NDVI | Среднее по пикселям в ячейке |
| `ndvi_std` | Разброс NDVI | Стандартное отклонение |
| `bare_soil_pct` | Доля голой почвы | Процент пикселей без растительности |

Рекомендуемые источники: Sentinel-2 (10 м), Landsat 8/9 (30 м).

### Процесс привязки

1. Загрузка растра в формате GeoTIFF
2. Перепроецирование в UTM Zone 37N (EPSG:32637) для согласованности с сеткой
3. Вычисление производных растров (уклоны, экспозиция, NDVI и т.д.)
4. Зональная статистика по каждому полигону ячейки с использованием библиотеки `rasterstats`

## 2.3 Привязка точечных данных (оползни, землетрясения)

Для точечных данных применяются три метода: **подсчёт в ячейке**, **расстояние до ближайшей точки**, **буферная статистика**.

### Признаки из исторических оползней

| Признак | Описание | Метод агрегации |
|---------|----------|-----------------|
| `ls_count` | Количество оползней в ячейке | Подсчёт точек внутри полигона |
| `ls_has_event` | Был ли оползень | 1 если count > 0, иначе 0 |
| `ls_nearest_m` | Расстояние до ближайшего оползня | Расстояние от центроида до ближайшей точки |
| `ls_density_5km` | Плотность в радиусе 5 км | Подсчёт в буфере / площадь буфера |

### Признаки из сейсмических данных

| Признак | Описание | Метод агрегации |
|---------|----------|-----------------|
| `eq_count_50km` | Количество землетрясений в радиусе 50 км | Подсчёт в буфере вокруг центроида |
| `eq_max_mag_50km` | Максимальная магнитуда в радиусе 50 км | Максимум в буфере |
| `eq_nearest_km` | Расстояние до ближайшего землетрясения | Расстояние от центроида |

### Процесс привязки

1. Загрузка точек с координатами (широта/долгота)
2. Конвертация в GeoDataFrame, перепроецирование в UTM Zone 37N
3. **Подсчёт в ячейке:** пространственное соединение (`sjoin` с предикатом `within`)
4. **Расстояние:** построение KD-дерева (`scipy.spatial.cKDTree`) для быстрого поиска ближайшей точки от каждого центроида
5. **Буферный анализ:** создание буфера заданного радиуса вокруг центроида, подсчёт точек внутри буфера

### Преимущество H3 для точечных данных

H3 позволяет эффективно привязывать точки к ячейкам через функцию `h3.latlng_to_cell()`, которая мгновенно определяет cell_id для любой точки по её координатам. Это значительно быстрее, чем пространственное соединение полигонов.

## 2.4 Привязка табличных/временных данных (погодные данные)

Для данных, имеющих временную привязку, но не имеющих пространственной детализации на уровне ячейки, применяется **пространственная интерполяция** от ближайших метеостанций или растровых продуктов.

### Признаки из погодных данных

| Признак | Описание | Метод агрегации |
|---------|----------|-----------------|
| `precip_7d` | Осадки за 7 дней | Интерполяция на центроид |
| `precip_30d` | Осадки за 30 дней | Интерполяция на центроид |
| `precip_90d` | Осадки за 90 дней | Интерполяция на центроид |

Рекомендуемые источники: Росгидромет, ERA5, CHIRPS, WorldClim.

### Процесс привязки

1. Загрузка данных о осадках (временные ряды по станциям или растр)
2. Пространственная интерполяция (IDW, Kriging, или извлечение из растра) на координаты центроидов ячеек
3. Агрегация за периоды 7, 30, 90 дней
4. Присвоение значений каждой ячейке
